## Task 21 — S2 storyboarding gate

**What it does.** After the model produces `outline.json`, print all slide titles in order and require the operator to confirm they read them aloud by setting `read_aloud_test: true` in `outline.json` before S3 can proceed.

**Acceptance criteria.**
- `gate_check_storyboard.py` exists under `references/scripts/`.
- Running it against an `outline.json` with `read_aloud_test: false` or the field missing exits non-zero and prints a human-readable error listing all slide titles.
- Running it against an `outline.json` with `read_aloud_test: true` exits 0.
- SKILL.md S2 section references the gate and blocks progression to S3 until it passes.
- One test in `tests/` covers the pass case and one covers the fail case.

## Task 22 — Auto-fix mode on render gate

**What it does.** For known minor overflow patterns (text overflow ≤ 50% of box, body overflow ≤ 0.3"), the render gate shrinks font by 1pt or trims 5 trailing chars and re-renders once. Caps at 1 retry. Every auto-fix is logged loudly in the gate output.

**Acceptance criteria.**
- `gate_check_render.py --auto-fix` accepts the flag without error.
- A deck with exactly one minor text overflow passes `--auto-fix` without operator intervention; gate output flags the auto-fix with the slide number, field, and action taken.
- A deck with a severe overflow (> 50%) fails even with `--auto-fix`; error message distinguishes severe from auto-fixable.
- No auto-fix runs without the flag; baseline behavior is unchanged.
- One test covers the auto-fix success case; one covers the severe-overflow rejection.

## Task 23 — Stronger section_divider

**What it does.** Redesigns `section_divider` so it reads visibly different from surrounding content slides in a long deck.

**Target layout.** Top accent bar (full width, forest green) + 72pt bold numeral (centered, forest green) + 32pt title (centered, dark) + 14pt italic subtitle (centered, muted). No content area. No source line.

**Acceptance criteria.**
- `eng.section_divider(number, title, subtitle)` renders with the above structure.
- A 30-slide deck with 3 section dividers: each divider is visually distinct from adjacent content slides (no body text box, no source line, accent bar present).
- Render gate passes on the updated layout.
- Schema entry in `api-schemas.yaml` updated; cheatsheet regenerated.
- Existing `section_divider` test updated or replaced; passes.
